ServiceContainerInterface

interface ServiceContainerInterface

A simple service container interface

The "register" system will store a callback in the registry with a specific name to use when trying to get a service. The getter will throw an error by default if the service does not exist, except when using the FAIL_GRACEFULLY constant. Any service is overwritable by default but can be not using the $overwritable = false register argument.

Constants

FAIL_GRACEFULLY

Use this constant to NOT throw error when trying to get an unknown service
FAIL_WITH_ERROR

Use this constant to throw error when trying to get an unknown service

This is the default behavior.
